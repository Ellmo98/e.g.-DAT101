    Discussion Questions:
  1. 1. What is the difference between invoking and creating a function?

    2. Creating a function means defining it and giving it a name, parameters, and a code block. The function is not executed when it is created. Invoking a function means calling it by its name followed by parentheses, which causes the code inside the function to run.

  2. 2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons are used to end JavaScript statements, but they are not used after function declarations or code blocks. Code blocks are defined using curly braces {} and contain the statements that belong to a function, loop, or condition.

  3. 3. What happens when we try to store values from a function when it doesn't return any?

    4. If a function does not return a value, JavaScript automatically returns undefined. Storing the result of such a function in a variable will give that variable the value undefined.

  4. 4. How do we separate parameters from each other when creating a function?

    5. Parameters are separated using commas inside the parentheses of the function definition. Example: function example(param1, param2, param3) {}

  5. 5. Explain how parameter names are related during the invocation and creation of a function.

    6. The parameter names used when creating a function act as placeholders. When invoking the function, the values passed in are assigned to these parameters in the same order, regardless of the variable names used during the call.

  6. 6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. If a function is called with fewer arguments than parameters, the missing parameters are automatically set to undefined by JavaScript.

  7. 7. Explain what happens when we use recursive functions.

    8. A recursive function calls itself to solve a problem. To avoid infinite loops, it must include a stopping condition (base case). Each call reduces the problem until the base case is reached, after which the function returns values back through the call stack.
